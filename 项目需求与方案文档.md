# 记分板 — 项目需求与方案文档

> 更新时间：2026-06-23 | 版本：v3.0 | 分支：master

---

## 一、项目概述

**定位**：单文件离线 HTML 记分板，面向羽毛球等对抗性运动的实时记分场景。免安装、免联网，手机浏览器 / 微信直接打开，全屏横屏为核心使用形态。

**仓库**：https://github.com/peter-J001/scoreboard  
**线上地址**：https://peter-J001.github.io/scoreboard/

**技术栈**：纯 HTML/CSS/JS，零依赖，单文件 ~21KB（压缩前）。

---

## 二、核心功能清单

### 2.1 记分交互

| 功能 | 说明 |
|------|------|
| 滑动加分 | 上滑 +1，下滑 -1（统一上下滑，避免横屏误触浏览器侧滑返回） |
| 防误触 | 滑动阈值 20px，小于阈值视为点击不触发 |
| 分数下限 | 不低于 0，无负数 |
| 分数动画 | 0.15s 弹跳动画（scale 1→1.08→1） |
| 联动同步 | 主界面与全屏分数实时联动 |

### 2.2 全屏模式（核心场景）

| 特性 | 说明 |
|------|------|
| 竖屏全屏 | 蓝队左、红队右，上下滑计分，队名隐藏（颜色区分），分数自适应放大 |
| 横屏全屏 | 同上布局，分数最大化利用屏幕高度 |
| 分数尺寸 | `clamp(90px, min(28vw, 75vh), 380px)` — 横竖屏自适应，不溢出 |
| 进入方式 | 点击「全屏计分」按钮，调用浏览器原生 Fullscreen API |
| 退出方式 | 点击黑色空白背景区域退出（滑动不会误退出） |
| 面板隐藏 | 全屏时自动隐藏功能面板和底部按钮 |
| 时间显示 | 保留顶部时间，全屏可读 |

### 2.3 可折叠功能面板

右下角「≡」按钮展开/收起，包含低频操作：

- **对局管理**：保存本场记录、新建对局清零
- **记录操作**：复制记录文本（纯文本格式）
- **分享**：发送页面文件（Web Share API / 下载 HTML，支持 AirDrop / 附近分享）
- **历史记录**：按时间倒序展示、单条删除、一键清空
- **特效开关**：获胜彩带与震动（默认开启）

### 2.4 数据持久化

| 功能 | 实现 |
|------|------|
| 当前分数 | localStorage 自动保存，刷新/重启不丢失 |
| 历史记录 | localStorage 数组存储，含 id / datetime / blueScore / redScore |
| 格式校验 | 导入数据校验，格式错误弹出提示不崩溃 |

### 2.5 羽毛球获胜庆祝（v3.0 新增）

**判定规则**（内置固定）：
1. 得分 ≥21 且领先 ≥2 分 → 获胜
2. 双方 ≥20 分后，先到 30 分直接获胜（封顶）

**触发效果**：
- Canvas 全屏彩带粒子动画（5 秒），获胜方主色调（蓝/红）+ 金色辅色
- 震动序列 `[100, 50, 100, 50, 100]`（不支持震动的设备自动跳过）
- 彩带纯视觉叠加（`pointer-events: none`），不影响滑动计分

**开关控制**：面板内「获胜彩带与震动」开关，默认开启；关闭后不判断胜负。

**重要约束**：
- 不自动保存、不自动清零，全部手动操作
- 不受单次限制，每次满足条件都触发
- 代码与计分逻辑解耦，删除不影响核心功能

## 四、技术方案

### 4.1 文件结构

```
index.html          # 单文件，全部 HTML/CSS/JS
LICENSE             # MIT
项目初始需求.txt     # 原始需求文档
项目需求与方案文档.md  # 本文档
```

### 4.2 CSS 架构

- **基础样式**：深色黑底 (#000)，红 (#EF4444) / 蓝 (#3B82F6) 高对比，系统字体栈
- **布局**：Flexbox 纵向，分数区 flex:1 填充剩余空间
- **全屏模式**：`.fs-mode` class 控制，隐藏非必要元素，覆盖分数/标签尺寸
- **自适应**：`clamp()` + `min()` 实现横竖屏自适配，无需 JS 监听 orientation
- **禁止缩放**：`user-scalable=no, maximum-scale=1.0`

### 4.3 JS 模块划分

| 模块 | 函数 | 说明 |
|------|------|------|
| State | `state` 对象 | 集中管理 blueScore / redScore / isFullscreen / isPanelOpen / confettiEnabled |
| Storage | `saveState / loadState / loadRecords / saveRecords` | localStorage 读写，try-catch 保护 |
| UI | `updateScoreDisplay / animateScore / updateTime / renderHistory` | DOM 更新 |
| Swipe | `initSwipe / getSwipeDir` | 触摸事件，20px 阈值，垂直优先 |
| Fullscreen | `requestFS / exitFS / isFS / handleFSChange` | Fullscreen API + webkit 兼容 |
| Panel | `openPanel / closePanel / togglePanel` | 面板动画控制 |
| Actions | `resetGame / newGame / saveRecord / copyText / sharePage / deleteRecord / clearAllRecords` | 业务操作 |
| Victory | `checkVictory / vibrateWin / startConfetti` | 羽毛球胜负判定 + 庆祝特效 |
| Toast | `showToast` | 2 秒自动消失提示（success/error） |

### 4.4 关键设计决策

1. **统一上下滑**：不再区分左右/上下滑动，全局上滑+1 下滑-1。原因：横屏左右滑触发浏览器返回，上下滑无冲突
2. **全屏队名隐藏**：颜色已足够区分队伍，文字冗余，隐藏后可释放垂直空间给分数
3. **`min(28vw, 75vh)` 分数尺寸**：竖屏宽受限用 vw，横屏高受限用 vh，自动跟随旋转
4. **Canvas 在 #app 内部**：全屏元素需在 fullscreen 容器内才能渲染
5. **彩带 pointer-events: none**：不拦截触摸事件，计分操作不受影响
6. **localStorage 仅存分数**：confettiEnabled 等 UI 状态不持久化，保持简单

### 4.5 浏览器兼容

- iOS Safari（WebKit Fullscreen API）
- Android Chrome
- 微信内置浏览器（功能完整）
- 桌面 Chrome / Edge / Firefox
